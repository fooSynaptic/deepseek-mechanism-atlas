# Markdown 公式 LaTeX 约定

在 **论文导读、Wiki、qa 答疑** 等 Markdown 中写公式时，必须保证 **Cursor 预览 / GitHub GFM** 可渲染。

## 分隔符（最重要）

| 场景 | ✅ 使用 | ❌ 禁止 |
|------|--------|--------|
| 行内公式 | `$...$` | `\(...\)`、纯括号 `(x_1,\ldots,x_T)` |
| 独立公式 | 前后各空一行的 `$$...$$` | `\[...\]`、无空行紧贴的 `$$` |

**原因**：`\(...\)` / `\[...\]` 在 Cursor 与 GitHub 默认预览中常**不渲染**，会原样显示为带反斜杠的括号文本（Engram 导读曾因此「公式挂了」）。

```markdown
<!-- ✅ -->
给定序列 $X=(x_1,\ldots,x_T)$，在第 $\ell$ 层 hidden $h_t^{(\ell)}$ 处：

$$
\alpha_t = \sigma\left(\frac{\mathrm{RMSNorm}(h_t)^{\top} \mathrm{RMSNorm}(k_t)}{\sqrt{d}}\right)
$$

<!-- ❌ -->
给定序列 \(X=(x_1,\ldots,x_T)\)
\[
\alpha_t = \sigma(...)
\]
```

## KaTeX 兼容写法

| 问题 | ✅ 推荐 | ❌ 易坏 |
|------|--------|--------|
| 带撇下标 | `$x^{\prime}_t$`、`$x^{\prime}_{t-n+1}$` | `$x_t'$`（解析歧义） |
| 转置 | `$h^{\top}$` | `$h^\top$`（复杂式中慎用） |
| 正体下标 | `$d_{\mathrm{mem}}$` | `$d_{\text{mem}}$`（部分渲染器不支持 `\text`） |
| 连缀拼接 | `$\mathop{\Big\Vert}\limits_{n=2}^{N}$` | `$\Vert_{n=2}^{N}$`（易被当成范数） |
| 长公式 | 拆成独立 `$$...$$` 块 | 单行 `$...$` 超过 ~80 字符 |
| 表格内公式 | 单元格内 `$...$` | 裸写 `\rho`、`W_K` 无 `$` |

## 符号与排版

- **禁止**用 Unicode 数学符号代替 LaTeX：`Δ α ρ ∈ → ≈ ×` → 用 `$\Delta$`、`$\alpha$`、`$\rho$`、`$\in$`、`$\to$`、`$\approx$`、`$\times$`
- **禁止**用代码块写公式（`` `W_0 x + ΔW x` ``）
- 分数用 `$\frac{a}{b}$`，不用 `α/r`
- 集合/空间用 `$\mathbb{R}^{d \times k}$`
- 行内公式两侧留空格：`在 $\ell$ 层`（非 `在$\ell$层`）

## 写入后自检

Agent 修改含公式的 `.md` 后，快速检查：

1. 全文无 `\(`、`\)`、`\[`、`\]` 作为数学分隔符
2. 每个 `$$` 块前后有空行
3. 预览中公式为渲染后的数学体，而非原始 `_`、`\ldots` 文本

## 反例（真实踩坑）

- ❌ `最优 $\rho \approx 80%$` 与同段 `\(...\)` 混用
- ❌ 独立公式写在列表项同一行且无 `$$` 空行
- ❌ `$e_t = \Vert_{n=2}^{N} \Vert_{k=1}^{K}$` 在预览中显示异常
- ❌ 用 `(X=(x_1,\ldots,x_T))` 想表达公式但未加 `$`
